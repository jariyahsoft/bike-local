# 1. Executive Summary

Bike Local คือแพลตฟอร์มศูนย์รวมร้านเช่าจักรยานและอุปกรณ์ปั่นจักรยานในแต่ละพื้นที่ ช่วยให้ผู้ใช้งานสามารถค้นหา เปรียบเทียบ จอง ชำระเงิน รับจักรยาน ติดตามกิจกรรมการปั่น ขอความช่วยเหลือ และคืนจักรยานผ่านแอปเดียว

ร้านเช่าจักรยานสามารถนำจักรยานและอุปกรณ์ของตนเข้าสู่ระบบ จัดการราคา การจอง การชำระเงิน พนักงาน จุดรับรถ จุดคืนรถ และติดตามผลประกอบการผ่านระบบหลังบ้าน

Bike Local ไม่ได้มีเป้าหมายเป็นเจ้าของจักรยานเอง แต่ทำหน้าที่เป็น Marketplace และระบบบริหารร้านเช่าในรูปแบบเดียวกับแพลตฟอร์มที่รวบรวมโรงแรม ร้านอาหาร หรือผู้ให้บริการท้องถิ่นไว้ในระบบเดียว

จุดเริ่มต้นของโครงการจะเน้นเฉพาะ

* จักรยาน
* จักรยานไฟฟ้า
* จักรยานสำหรับเด็ก
* จักรยานเพื่อการท่องเที่ยวและออกกำลังกาย
* อุปกรณ์ปั่นจักรยาน

ในอนาคตสามารถต่อยอดระบบโครงสร้างพื้นฐานเดิมไปสู่แพลตฟอร์มเช่ายานพาหนะอื่น เช่น มอเตอร์ไซค์ รถไฟฟ้าขนาดเล็ก และยานพาหนะเพื่อการท่องเที่ยว โดยแยกเป็นอีกแอปหนึ่งเพื่อลดความซับซ้อนของผลิตภัณฑ์หลัก

# 2. Vision

สร้างแพลตฟอร์มกลางที่ทำให้ผู้ใช้งานสามารถค้นหาและเช่าจักรยานจากร้านท้องถิ่นได้อย่างสะดวก ปลอดภัย และโปร่งใส พร้อมช่วยให้ร้านเช่าขนาดเล็กสามารถเข้าสู่ระบบดิจิทัลและเพิ่มรายได้โดยไม่ต้องพัฒนาระบบของตนเอง

# 3. Mission

* รวมร้านเช่าจักรยานท้องถิ่นไว้ในแอปเดียว
* ทำให้การค้นหาและจองจักรยานง่ายขึ้น
* เพิ่มช่องทางรายได้ให้ร้านค้าในพื้นที่
* ลดขั้นตอนการรับรถและคืนรถ
* ส่งเสริมการท่องเที่ยวและการออกกำลังกายด้วยจักรยาน
* สร้างข้อมูลเส้นทางและสถานที่ปั่นที่เชื่อถือได้
* วางโครงสร้างพื้นฐานสำหรับต่อยอดเป็นแพลตฟอร์มเช่ายานพาหนะในอนาคต

# 4. Problem Statement

## 4.1 ปัญหาของผู้เช่า

ผู้ที่ต้องการเช่าจักรยานมักพบปัญหา เช่น

* ไม่ทราบว่ามีร้านเช่าจักรยานอยู่ที่ใด
* ต้องค้นหาผ่านหลายช่องทาง
* ไม่มีข้อมูลราคาและจำนวนรถว่างที่ชัดเจน
* ไม่สามารถจองล่วงหน้าได้
* ไม่สามารถเปรียบเทียบร้านได้
* ขั้นตอนการจ่ายเงินและรับรถแตกต่างกันในแต่ละร้าน
* ไม่มีระบบบันทึกการปั่นในแอปเดียวกับระบบเช่า
* ไม่ทราบเส้นทางปั่นหรือจุดเช็กอินที่เหมาะสม
* เมื่อลูกค้าเกิดปัญหาระหว่างทาง ไม่มีระบบส่งพิกัดให้ร้านทันที
* การคืนจักรยานอาจต้องกลับไปยังร้านเดิมเท่านั้น

## 4.2 ปัญหาของร้านเช่า

ร้านเช่าจักรยานท้องถิ่นจำนวนมากยังบริหารงานผ่าน

* สมุดจด
* โทรศัพท์
* Facebook หรือ LINE
* ตาราง Excel
* การรับเงินสดโดยไม่มีระบบตรวจสอบ
* การจัดการจักรยานและอุปกรณ์แบบไม่เป็นระบบ

ร้านจึงประสบปัญหา เช่น

* เข้าถึงลูกค้าใหม่ได้ยาก
* ไม่สามารถแสดงจำนวนรถว่างแบบเรียลไทม์
* การจองซ้ำซ้อน
* ไม่สามารถติดตามงานของพนักงานได้
* ไม่ทราบรายได้จริงของรถแต่ละคัน
* ไม่มีรายงานการเช่าและการเงิน
* การรับและคืนจักรยานใช้เวลานาน
* ไม่มีระบบจัดการจุดคืนหลายจุด
* ไม่มีระบบรับแจ้งเหตุฉุกเฉินจากผู้เช่า

# 5. Product Positioning

Bike Local เป็น

> “Marketplace และระบบบริหารร้านเช่าจักรยานท้องถิ่น ที่ช่วยให้ผู้ใช้งานค้นหา จอง ปั่น และคืนจักรยานได้ในแอปเดียว”

ผลิตภัณฑ์ประกอบด้วย 3 ส่วนหลัก

1. แอปหรือเว็บไซต์สำหรับผู้เช่า
2. แอปหรือระบบสำหรับร้านและพนักงาน
3. ระบบผู้ดูแลแพลตฟอร์ม

* Wallet
* คูปอง
* เครดิตร้านค้า

## 16.2 การชำระเงินออนไลน์

เมื่อระบบยืนยันการชำระเงินสำเร็จ

1. เปลี่ยนสถานะเป็น “ชำระเงินแล้ว”
2. แจ้งเตือนร้านและพนักงาน
3. แสดงข้อมูลลูกค้า จักรยาน และจุดรับ
4. ให้พนักงานกดรับงาน
5. อนุญาตให้ส่งมอบหรือปลดล็อกจักรยาน
6. บันทึกประวัติธุรกรรม

## 16.3 การชำระเงินสด

เมื่อผู้ใช้เลือกเงินสด

1. รายการอยู่ในสถานะ “รอรับเงินสด”
2. ร้านหรือพนักงานได้รับแจ้งเตือน
3. พนักงานรับเงินและกดยืนยัน
4. ระบบบันทึกชื่อพนักงาน จำนวนเงิน และเวลา
5. จึงอนุญาตให้ส่งมอบจักรยาน

ร้านสามารถกำหนดได้ว่า

* รับหรือไม่รับเงินสด
* รับที่ร้านเท่านั้น
* รับที่จุดส่งมอบได้
* พนักงานคนใดรับเงินสดได้
* ต้องเก็บเงินมัดจำหรือไม่

## 16.4 เงินมัดจำ

ร้านสามารถกำหนด

* ไม่เก็บเงินมัดจำ
* มัดจำแบบคงที่
* มัดจำตามประเภทจักรยาน
* มัดจำตามระยะเวลาการเช่า
* เงินมัดจำเป็นเงินสดหรือออนไลน์

เงินมัดจำจะยังไม่คืนจนกว่าร้านหรือพนักงานจะยืนยันรับจักรยานคืนและตรวจสภาพแล้ว

# 32. Non-Functional Requirements

## 32.1 Performance

* ระบบจองและชำระเงินควรพร้อมใช้งานตลอดเวลา
* มีระบบสำรองข้อมูล
* มีระบบตรวจสอบสถานะบริการ
* เมื่อบริการบางส่วนล้มเหลว ต้องไม่ทำให้ข้อมูลธุรกรรมสูญหาย

## 32.3 Security

* เข้ารหัสข้อมูลระหว่างรับส่ง
* จัดเก็บข้อมูลสำคัญอย่างปลอดภัย

* บันทึก Audit Log
* จำกัดการดูข้อมูลพิกัด
* มีระบบป้องกันการเข้าถึงบัญชีโดยไม่ได้รับอนุญาต
* ไม่เก็บข้อมูลบัตรโดยตรง หากใช้ผู้ให้บริการชำระเงิน

## 32.4 Privacy

* ขอความยินยอมก่อนติดตามตำแหน่ง

## 32.5 Accessibility

# 36. Key User Flows

## 36.1 เช่าและชำระเงินออนไลน์

ค้นหาร้าน → เลือกจักรยาน → เลือกเวลา → เลือกจุดรับและคืน → ชำระเงิน → ร้านได้รับแจ้ง → พนักงานส่งมอบ → เริ่มปั่น → จบการปั่น → ส่งคำขอคืน → พนักงานรับคืน → ปิดรายการ

## 36.2 เช่าและจ่ายเงินสด

ค้นหา → จอง → เลือกเงินสด → ร้านยืนยัน → พนักงานรับเงิน → ยืนยันในระบบ → ส่งมอบ → เริ่มปั่น → คืน → ตรวจรับ → ปิดรายการ

## 36.3 ร้านมารับจักรยาน

ผู้ใช้จบการปั่น → เลือกบริการรับคืน → ส่งพิกัด → ร้านมอบหมายพนักงาน → พนักงานเดินทางไป → ตรวจรับ → ปิดรายการ

## 36.4 SOS

ผู้ใช้กด SOS → เลือกปัญหา → ส่งพิกัดและข้อมูลจักรยาน → ร้านได้รับแจ้ง → พนักงานรับเคส → ติดต่อผู้ใช้ → ให้ความช่วยเหลือ → ปิดเคส

# 37. Acceptance Criteria สำคัญ

## 37.1 การชำระเงิน

* เมื่อชำระออนไลน์สำเร็จ ร้านและพนักงานต้องได้รับแจ้งเตือน
* เงินสดต้องยืนยันโดยพนักงานที่มีสิทธิ์
* ระบบต้องเก็บประวัติผู้รับเงิน
* ห้ามเริ่มรายการเช่าก่อนยืนยันการชำระเงิน เว้นแต่ผู้มีสิทธิ์อนุมัติ

## 37.2 GPS ใน MVP

* ใช้ GPS จากโทรศัพท์ได้โดยไม่ต้องมี GPS ที่จักรยาน
* บันทึกจุดเริ่มและสิ้นสุด
* แสดงเส้นทางย้อนหลัง
* แจ้งเมื่อ GPS ถูกปิด
* ไม่สร้างข้อมูลตำแหน่งขึ้นเอง

## 37.3 จบการปั่นและคืนจักรยาน

* กดจบการปั่นแล้วต้องไม่ปิดรายการเช่าทันที
* เปลี่ยนสถานะเป็นรอยืนยันการคืน
* ผู้เช่ายังรับผิดชอบจักรยานจนกว่าจะยืนยันรับคืน
* ร้าน พนักงาน หรือ Smart Dock ต้องยืนยัน
* ยังไม่คืนเงินมัดจำก่อนยืนยัน
* บันทึกผู้ยืนยัน วันเวลา ตำแหน่ง และหลักฐาน

## 37.4 SOS

* ส่งตำแหน่งล่าสุดให้ร้านได้
* ร้านเห็นข้อมูลรายการเช่า
* พนักงานกดรับเคสได้
* มีประวัติเวลาตอบรับและปิดเคส

## 37.5 สถานที่และเส้นทาง

* ข้อมูลจากสมาชิกไม่เผยแพร่ก่อนอนุมัติ
* ต้องระบุประเภทจักรยานที่เหมาะสม
* ผู้ใช้รายงานข้อมูลผิดหรือไม่ปลอดภัยได้

# 39. Pilot Launch Recommendation

ควรเริ่มโครงการนำร่องในหนึ่งพื้นที่ก่อน โดยมีเป้าหมายเบื้องต้น เช่น

* ร้านเข้าร่วม 10–30 ร้าน
* จักรยาน 100–500 คัน
* จุดรับและคืน 20–50 จุด
* ผู้ใช้งานกลุ่มแรก 1,000–5,000 คน
* ระยะเวลาทดลอง 3–6 เดือน

สิ่งที่ต้องประเมินระหว่าง Pilot

* ผู้ใช้ค้นหาร้านและจองได้ง่ายหรือไม่
* ร้านใช้งานระบบหลังบ้านได้หรือไม่
* การจ่ายเงินสดและออนไลน์มีปัญหาใด
* การแจ้งเตือนพนักงานรวดเร็วหรือไม่
* การรับคืนต่างจุดมีต้นทุนเท่าใด
* GPS จากโทรศัพท์มีความแม่นยำเพียงพอหรือไม่
* ร้านตอบรับ SOS ได้เร็วเพียงใด
* โมเดลค่าคอมมิชชันเหมาะสมหรือไม่

# 40. Key Risks and Mitigation

## 40.1 ร้านไม่อัปเดตจำนวนรถ

แนวทางแก้ไข

* เปลี่ยนสถานะอัตโนมัติตามการจอง
* แจ้งเตือนร้านตรวจสอบสต็อก
* มีระบบปิดรถที่ไม่พร้อมให้เช่า

## 40.2 ผู้ใช้กดจบการปั่นแต่ไม่คืนรถ

แนวทางแก้ไข

* ไม่ปิดรายการจนร้านยืนยัน
* ไม่คืนเงินมัดจำ
* ขอรูปและพิกัด
* แจ้งเตือนร้านและผู้ใช้

## 40.3 เงินสดไม่ตรงกับรายงาน

แนวทางแก้ไข

* จำกัดสิทธิ์รับเงิน
* บันทึกพนักงานผู้รับ
* รายงานเงินสดแยกตามกะและพนักงาน
* Audit Log

## 40.4 GPS โทรศัพท์ไม่ต่อเนื่อง

แนวทางแก้ไข

* แจ้งเตือนเมื่อปิด GPS
* เก็บตำแหน่งล่าสุด
* แสดงช่วงข้อมูลขาด
* เพิ่ม GPS ที่จักรยานใน Phase 2

## 40.5 ร้านตอบ SOS ช้า

แนวทางแก้ไข

* แจ้งเตือนหลายระดับ
* ส่งต่อถึงเจ้าของร้าน
* กำหนด SLA
* แสดงหมายเลขฉุกเฉิน

## 40.6 ข้อมูลเส้นทางไม่ถูกต้อง

แนวทางแก้ไข

* ใช้ระบบอนุมัติ
* มีวันทบทวนข้อมูล
* ให้ผู้ใช้รายงาน
* ระงับเส้นทางที่ถูกรายงานหลายครั้ง

# 41. Competitive Advantages

* รวมร้านท้องถิ่นหลายร้านไว้ในแอปเดียว
* ร้านไม่ต้องลงทุนพัฒนาระบบเอง
* รองรับทั้งเงินสดและออนไลน์
* มีระบบพนักงานและหลายสาขา
* รองรับจุดคืนหลายจุด
* มีข้อมูลกิจกรรมออกกำลังกายในแอป
* มี SOS ส่งพิกัดให้ร้าน
* มีเส้นทางและสถานที่แนะนำ
* ไม่ต้องลงทุนซื้อจักรยานเอง
* สามารถต่อยอด Smart Lock, GPS และแพลตฟอร์มยานพาหนะได้

# 42. Conclusion

Bike Local มีศักยภาพเป็นโครงสร้างพื้นฐานดิจิทัลสำหรับธุรกิจเช่าจักรยานท้องถิ่น โดยเชื่อมโยงผู้เช่า ร้านค้า พนักงาน จุดท่องเที่ยว และเส้นทางปั่นไว้ในระบบเดียว

แนวทางเริ่มต้นด้วยจักรยานและอุปกรณ์ช่วยลดความซับซ้อน ความเสี่ยงด้านกฎหมาย และต้นทุนการพัฒนา เมื่อระบบมีฐานร้านค้าและผู้ใช้งานเพียงพอ จึงสามารถต่อยอดสู่แอปเช่ายานพาหนะประเภทอื่นโดยใช้ระบบส่วนกลางร่วมกัน

เป้าหมายของ Bike Local ไม่ใช่เพียงการสร้างแอปเช่าจักรยาน แต่คือการสร้าง Marketplace และระบบบริหารธุรกิจเช่าท้องถิ่นที่สามารถขยายได้ในระยะยาว
